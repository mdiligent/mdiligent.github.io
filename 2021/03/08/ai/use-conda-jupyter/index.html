常见问题

问题：Anaconda：WARNING: The conda.compat module is deprecated and will be removed in a future release.
原因：清华源出现问题或者conda需要更新。
解决办法：换源，更新condaconda update conda

问题：今天突然又不能用了，我重置设置成同样的又能用了，不知道是不是以为我自己的网络问题，还是配置不对。

Jupyter切换虚拟环境

1.创建一个虚拟环境。
2.激活某个虚拟环境

3.在此环境中中安装好ipykernel

4.将此环境（也可以是别的环境）添加到ipython的kernel中，只输入<>块之前的就可

删除kernel中的环境

  1. 查看当前的kernel及其路径的命令

  2. 进入那个文件夹把对应目录删除

注意：要删除虚拟环境，需要先删除kernel。

https://blog.csdn.net/Junhao_Lu/article/details/100625831

可能遇见的问题：

  • ImportError: cannot import name ‘generator_to_async_generator’

  • ModuleNotFoundError: No module named ‘IPython.core’

  • 打开conda管理环境报错EnvironmentLocationNotFound: Not a conda environment: F:\Anaconda3\envs\Anaconda3

    解决方法：https://www.jianshu.com/p/7d8ce818a923，因为我把一个虚拟环境设置了环境变量，所以启动的jupyter是虚拟环境中的，所以修改文件需要去修改虚拟环境目录的文件。

Jupyter 重装

1.手动删除所有包

2.再手动删除Lib/site-packges和Scripts下所有jupyter相关的文件

3.重新安装

参考：https://blog.csdn.net/sheqianweilong/article/details/89843250

Jupyter 使用技巧

  • 更改工作目录
    需知：在命令行中打开jupyter，默认工作目录时安装目录，除非cd到指定路径(或者在某个文件夹右键)打开，但是指定了工作目录，就没法打开指定目录了，可以自行斟酌，根据需求取舍。 设置工作目录具体步骤： 在系统盘，用户名文件夹下的.jupyter文件夹下修改jupyter_notebook_config.py，修改c.NotebookApp.notebook_dir的值。

常识

conda install 是为所激活的虚拟环境安装包，而不是所有虚拟环境共享。

因为我设置了环境变量，所以每次我启动jupyter时，都是启动虚拟环境中的，而不是conda的默认base环境中的。

使用conda install其实包都安装到激活的环境中去了。
